## Trade Service

The Trade Service is responsible for order matching and trade execution within the broker platform, acting as the system’s matching engine. It consumes order-placed events from Kafka, maintains an in-memory order book per stock symbol using concurrent data structures, and continuously matches BUY and SELL orders based on price-time priority, executing trades when the highest buy price meets or exceeds the lowest sell price. Each successful match generates paired BUY and SELL trade records with a shared trade identifier, which are persisted in a dedicated PostgreSQL database using a composite key and published as trade-placed events to Kafka for downstream processing. The service exposes secured endpoints to query and purge trades, runs independently of synchronous dependencies, and is designed to be low-latency, event-driven, and horizontally scalable, closely mirroring real-world exchange trade execution workflows.
