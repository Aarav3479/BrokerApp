## User Service

The User Service acts as the authentication and identity authority of the broker platform, responsible for user registration, login, profile management, and deletion. It implements stateless JWT-based authentication using Spring Security, where users authenticate via email and password, credentials are validated through a DaoAuthenticationProvider, and a JWT (HS256, 1-hour expiry) is issued upon successful login. Passwords are securely stored using BCrypt hashing, and all protected endpoints require a valid Authorization: Bearer <token> header, enforced via a custom JWT filter integrated into the Spring Security filter chain. The service persists user data in a dedicated PostgreSQL database, registers itself with Eureka service discovery, and publishes user lifecycle domain events (user-created, user-deleted) to Kafka, enabling downstream services to react asynchronously in an event-driven manner. The service is fully Dockerized, runs on port 8081, and is orchestrated locally using Docker Compose alongside Kafka, PostgreSQL, and other platform services.
