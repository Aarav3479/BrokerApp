## Portfolio Service

The Portfolio Service is responsible for maintaining each user’s investment state, including stock holdings, quantities, average prices, and total portfolio value. It operates primarily in an event-driven manner, consuming user-created and user-deleted events from Kafka to automatically create or remove portfolios, and trade-placed events to atomically update portfolio positions after trade execution. For each trade, the service adjusts stock quantities, recalculates weighted average prices, updates total portfolio value, and removes positions when quantities drop to zero, all within transactional boundaries to ensure consistency. It also exposes secured REST endpoints to fetch portfolios by ID or email, list all portfolios, and perform controlled manual portfolio updates when required (used by the Order Service for sell-order validation). Portfolio data is persisted in a dedicated PostgreSQL database using a normalized portfolio–stock model, the service is registered with Eureka, fully Dockerized, and acts as the system’s source of truth for user holdings.
