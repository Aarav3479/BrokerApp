## Order Service

The Order Service is responsible for handling the complete order lifecycle within the broker platform, including order placement, retrieval, and deletion. All endpoints are secured using stateless JWT-based authentication, enforced via Spring Security and a custom JWT filter, ensuring only authenticated users can interact with order operations. When an order is placed, the service performs a synchronous validation for SELL orders by calling the Portfolio Service through a Feign client, verifying sufficient stock quantity before proceeding; this call is protected using a Resilience4j circuit breaker to prevent cascading failures. Upon successful validation, the order is persisted in a dedicated PostgreSQL database, and an order-placed domain event is published to Kafka, enabling asynchronous downstream processing by the Trade Service. The Order Service also consumes user-related events to stay aligned with system state, is registered with Eureka service discovery, runs on port 8082, and is fully Dockerized and orchestrated via Docker Compose as part of the overall event-driven microservices architecture.
